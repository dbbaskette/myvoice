# Format Add-On: Conference Abstract

Layer this on top of the master style guide for CFP submissions.

## Length

150–200 words. Most CFP forms enforce this. Hit the upper end.

## Structure

1. **The problem (1-2 sentences).** Conflict & Resolution opening, compressed.
2. **The approach (3-4 sentences).** What you'll show. Be specific about technologies.
3. **The demo (1-2 sentences).** What runs live on stage. "No smoke and mirrors."
4. **The takeaway (1 sentence).** Start with: *"Attendees will leave with…"*

## Required elements

- Name at least one specific tool, platform, or framework.
- Name the audience by role ("platform engineers," "Java developers," "data teams").
- Include the time-to-value number if there is one.

## Forbidden in abstract format

- "This talk will cover…" openings.
- Promising "deep dive" without saying into what.
- Buzzword stacking ("AI-powered cloud-native serverless platform").
- Using the word "exciting."
